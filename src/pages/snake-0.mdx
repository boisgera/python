## Pixi

The [Pixi] package manager can handle all your project software dependencies. 
It is a good choice for Python projects in general, and especially for hybrid
projects that require both Python packages and non-Python software, since 
it can install both Python packages from [PyPi] and Python *and* non-Python 
software from [conda-forge].

[Pixi]: https://prefix.dev/docs/pixi/overview
[conda-forge]: https://conda-forge.org
[PyPi]: https://pypi.org

<Admonition type="note" title="Pixi worflow" open={debug}>

[Install pixi](https://pixi.prefix.dev/latest/installation/), 
then for each new project, create a new empty pixi environment

```bash
pixi my_new_project
```

Then get into the newly created directory `my_new_project` 

```bash
cd my_new_project
```

and install what you need 
from conda-forge, for example

```bash
pixi install python
```
then if you also need packages from PyPi, like pyxel, install them with

```bash
pixi install --pypi pyxel
```
Develop your project with your favorite editor, for example Visual Studio Code,
and when you need to run your code, just type

```bash
pixi run python my_app.py
```

</Admonition>

An good alternative to Pixi for pure Python projects is [uv].

[uv]: https://docs.astral.sh/uv/

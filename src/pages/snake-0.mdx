# The Hard Way

Useful if you want:

  - to develop Python projects other than Pyxel games,

  - to work locally on your computer and not depend on "the cloud",

  - to benefit from a more feature-rich development environment.


## Visual Studio Code

  - Install [Visual Studio Code][VS-Code], then

  - Install the Microsoft [Python extension for Visual Studio Code][Python-VS-Code].

## Pixi

The [Pixi] package manager can help you to handle Python environments and 
dependencies, especially if you need some of the Python packages of [PyPi]
but also some of the packages of [conda-forge] (which include both Python and 
non-Python software).

[Pixi]: https://prefix.dev/docs/pixi/overview
[conda-forge]: https://conda-forge.org
[PyPi]: https://pypi.org

<Admonition type="note" title="Pixi worflow" open={debug}>

[Install pixi](https://pixi.prefix.dev/latest/installation/), 
then for each new project, create a new empty pixi environment

```bash
pixi my_new_project
```

Then get into the newly created directory `my_new_project` 

```bash
cd my_new_project
```

and install what you need 
from conda-forge, for example

```bash
pixi install python
```
then if you also need packages from PyPi, like pyxel, install them with

```bash
pixi install --pypi pyxel
```
Develop your project with your favorite editor, for example Visual Studio Code,
and when you need to run your code, just type

```bash
pixi run python my_app.py
```

</Admonition>

An good alternative to Pixi for pure Python projects is [uv].

[uv]: https://docs.astral.sh/uv/

## Troubleshooting

Common issues that you may have with "the hard way":

<Admonition type="warning" title="Pyxel is not found" open={debug}>
You have installed Pyxel with pip but Visual Studio Code cannot find it?
It is likely not using the same Python environment as the one
you used to install Pyxel.

  - In your terminal, type:
  
    ```bash
    python -c "import sys; print(sys.executable)"
    ```
 
    to know to which Python interpreter `python` refers to. 
   
  - In Visual Studio Code, open the command palette with `Ctrl+Shift+P` 
    and type `Python: Select Interpreter` to select this Python interpreter.

  - Install explicitly pyxel for this interpreter with:

    ```bash
    python -m pip install pyxel
    ```

    (your `pip` executable may refer to another Python interpreter.)

</Admonition>

<Admonition type="warning" title="undefined symbol: SDL_Metal_DestroyView" open={debug}>
